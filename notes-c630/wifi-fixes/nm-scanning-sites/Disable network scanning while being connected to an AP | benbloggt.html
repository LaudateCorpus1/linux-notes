Disable network scanning while being connected to an AP

1 Antwort

I got myself a brandnew Asus PCE-AC55BT. This seems to be the only avaiable wireless lan card, which is supported in Linux out of the box. Its Intel Corporation Wireless 7260 is supported directly through the iwlwifi kernel module.

But due to a Bug in the firmware the card doesn’t cope well with background scanning for networks. In consequence one will see lag spikes. This particularly annoying for online gaming: The game will periodically have those micro freeze, pumping pings up to ridiculous numbers. Those can also be see by just pinging an arbitrary host over a short period of time.

Since Intel won’t fix this only for new card and not for old ones (bought it yesterday!), the only solution is to disable scanning for network while a connection is already established to an AP.

Using NetworkManager this is done by locking the BSSID:

  1. Open NetworkManager GUI
  2. Click the gear icon next to the SSID
  3. Click on the Identity tab
  4. Select or type the BSSID in the BSSID dropdown
  5. Click Apply

It maybe worth following the development of NetworkMangager, as following upcoming features look promising:

  • Bug 766482 – [review] dcbw/wifi-scan: delegate periodic scanning behavior to wpa_supplicant and reduce periodic scans
  • Bug 778152 – Add option to disable scanning while connected

Cheers
